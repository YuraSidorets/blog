Hi, I'm Yurii.

I'm a .NET engineer based in Prague. Most of my work has been around backend systems, integrations, and the kind of enterprise software that becomes interesting only after it starts carrying real load, real business rules, and real operational pain.

Over the last several years I have worked on:

- large internal platforms and legacy migrations
- cloud services and web APIs
- CI/CD, observability, and developer workflow improvements
- data access and integration-heavy systems
- testing infrastructure, from integration tests to load testing

Most of that work has been in the .NET ecosystem, with PostgreSQL, SQL Server, Kafka, Docker, Kubernetes, GitHub Actions, and the usual set of APIs, queues, and background processes around it.

I currently work as a senior software engineer. Before that I worked across consulting, product development, and R&D teams, including projects in marketing technology, e-commerce, ERP-style business systems, and medical software.

This blog is intentionally small. The goal is not to publish polished thought leadership every week. It is to keep a useful set of technical notes, working examples, and opinions that came out of real engineering work.

## What you'll find here

Some posts are short notes. Some are implementation write-ups. The common thread is that they come from things I had to build, debug, migrate, automate, or explain properly at work.

That usually means posts about:

- .NET architecture and backend design
- testing, debugging, and performance work
- GitHub Actions, CI/CD, and workflow automation
- integration problems and distributed system edges
- tools, books, and material that were actually useful

## Languages

The blog is bilingual:

- [English](/english/) is the default language
- [Українською](/ukrainian/) contains the Ukrainian posts and translations

When a post exists in both languages, there is a language switcher directly on the post page.
